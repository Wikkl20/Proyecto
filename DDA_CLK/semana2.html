Introducción a la Semana 2

Domina condicionales en Java y crea apps más inteligentes.

En esta segunda semana se trabajaron problemas aplicando estructuras condicionales y validación de datos en Java con Apache NetBeans. Los ejercicios refuerzan el diseño modular, dividiendo el proyecto en paquetes (Main e IUG), y asegurando que la interacción con el usuario sea clara y segura mediante validaciones.

Actividad 01: Tienda de Docenas

Se calcula el importe de la compra aplicando descuentos según las docenas adquiridas: 20% si son 10 o más, o 10% si son menos. Además, si el importe ≥ S/.200, se otorgan 2 lapiceros por cada docena. El sistema valida entradas correctas y muestra importe total, descuento, pago final y obsequios.

Diagrama de Clases — Problema 1


class Producto {
  - precioDocena: double
  - cantidad: int
  + calcularImporte(): double
  + calcularDescuento(): double
  + calcularPago(): double
  + calcularLapiceros(): int
}

class Tienda {
  + vender(p: Producto)
}

El diseño encapsula los cálculos en la clase Producto, incluyendo importe, descuentos y obsequios. La clase Tienda representa el proceso de venta, utilizando los datos del producto.

Actividad 02: Sueldo de Empleado

Se calcula el sueldo de un empleado considerando: Sueldo básico (horas × tarifa según categoría), bonificación por hijos (S/.40.5 hasta 3 hijos, luego S/.35), y descuento por ley (13.5% si ≥ 3500, caso contrario 10%). El programa presenta sueldo básico, bruto, descuento y sueldo neto.

Diagrama de Clases — Problema 2


class Empleado {
  - categoria: char
  - horasTrabajadas: int
  - numHijos: int
  + calcularSueldoBasico(): double
  + calcularBonificacion(): double
  + calcularSueldoBruto(): double
  + calcularDescuento(): double
  + calcularSueldoNeto(): double
}

class Empresa {
  + procesarEmpleado(e: Empleado)
}

La clase Empleado gestiona todos los cálculos salariales: sueldo básico, bonificación, descuento y sueldo neto. La clase Empresa puede administrar múltiples empleados para procesar sus pagos.
